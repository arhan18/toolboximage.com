## When Do You Need to Rotate Images?

### 1. Wrong Camera Orientation
Your phone's orientation sensor doesn't always embed the correct EXIF rotation flag. The result: sideways photos that should be upright.

### 2. Scanning Documents
Scanners don't always detect page orientation. A 90° or 180° rotation fixes scanned PDF pages instantly.

### 3. Correcting Selfie Mirrors
Front-facing cameras on phones mirror images by default. A horizontal flip gives you the "true" image that others see.

### 4. Creating Mirror Compositions
Designers often flip images to create symmetrical compositions, mirror effects, or directional balance in layouts.

## The EXIF Orientation Problem

Modern cameras and phones embed a rotation flag in the image's EXIF metadata. When you view the image, your phone or computer reads this flag and automatically rotates the display.

The problem? Not all software respects EXIF orientation flags. Some tools:
- Ignore the flag entirely (showing the raw, unrotated image)
- Apply the flag inconsistently between preview and export
- Strip the flag during processing (permanently losing the orientation data)

**The ToolBox Image Rotator physically rotates the pixel data** — it doesn't rely on EXIF flags. When you rotate an image, every pixel is rearranged. The result is a permanently corrected image that displays correctly everywhere, regardless of software.

## Quick Tips for Best Results

1. **Rotate before cropping** — Rotating first gives you a better sense of the image's true orientation for subsequent edits.
2. **Use Flip H for text** — If you're mirroring an image that contains text, remember the text will be reversed.
3. **Custom angles add white space** — Rotating by non-90-degree angles will introduce white corners. The tool auto-crops to fit these.
4. **180° rotation is lossless** — Rotating by 180° doesn't require any pixel interpolation, so quality is perfectly preserved.
